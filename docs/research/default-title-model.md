# Research: Identify the default inexpensive title model

## Summary

Pi Prompt Title should **not hardcode one universal model**. The privacy/auth requirement (“only a provider already usable by the user may receive the prompt”) makes availability a runtime property: Pi exposes models that are available under configured credentials, and credentials are provider-scoped. The recommended policy is to choose the first exact provider/model pair available in this order: **`openai` / `gpt-5-nano`**, then **`google` / `gemini-2.5-flash-lite`**, then **`anthropic` / `claude-haiku-4-5`**; otherwise use an explicitly authenticated current-session model or do not generate a title. If a product decision nevertheless requires one named preferred candidate, use **`openai/gpt-5-nano`**: it has the lowest current list-price estimate and OpenAI explicitly describes it as very fast and suitable for summarization.

## Findings

1. **A fixed cross-user default conflicts with the provider-availability requirement.** Pi supports multiple subscription and API-key providers. API credentials are keyed by provider (`openai`, `google`, `anthropic`, etc.), and OpenAI Codex subscription access is separately described from OpenAI API-key access. Therefore, for example, a user authenticated only to `openai-codex`, GitHub Copilot, or Anthropic does not thereby authorize sending the title prompt to the `openai` API provider. Pi’s `ModelRegistry.getAvailable()`, `hasConfiguredAuth(model)`, and provider auth-status methods are the appropriate runtime gate; merely finding a model in the full catalog is insufficient. [Pi provider documentation](https://github.com/badlogic/pi-mono/blob/main/packages/coding-agent/docs/providers.md) [Pi `ModelRegistry` source](https://github.com/badlogic/pi-mono/blob/main/packages/coding-agent/src/core/model-registry.ts)

2. **`openai/gpt-5-nano` is the strongest named preference when that exact provider is usable.** OpenAI calls GPT-5 nano its “fastest, cheapest version of GPT-5,” rates speed “very fast,” and says it is good for summarization and classification—directly relevant capability signals for compressing a prompt into a short title. Its official price is $0.05/M input tokens and $0.40/M output tokens. Pi’s current official generated catalog contains the exact pair `provider: "openai"`, `id: "gpt-5-nano"` with matching prices and Responses API support. [OpenAI GPT-5 nano model card](https://platform.openai.com/docs/models/gpt-5-nano) [Pi OpenAI catalog](https://github.com/badlogic/pi-mono/blob/main/packages/ai/src/providers/openai.models.ts)

3. **`google/gemini-2.5-flash-lite` is a credible second preference, not a universal default.** Google describes it as its smallest, most cost-effective model built for at-scale use. The official standard paid price is $0.10/M text input and $0.40/M output (including thinking tokens), and the Pi catalog contains the exact stable-looking ID under provider `google`. It is twice GPT-5 nano’s input price under the comparison assumptions, while output price is equal. [Google Gemini API pricing](https://ai.google.dev/gemini-api/docs/pricing) [Google Gemini 2.5 Flash-Lite model page](https://ai.google.dev/gemini-api/docs/models/gemini-2.5-flash-lite) [Pi Google catalog](https://github.com/badlogic/pi-mono/blob/main/packages/ai/src/providers/google.models.ts)

4. **`anthropic/claude-haiku-4-5` is the quality/latency fallback for Anthropic-authorized users, but it is materially more expensive.** Anthropic calls Haiku 4.5 its fastest model with near-frontier intelligence, lists it as the fastest current Claude in comparative latency, and prices it at $1/M input and $5/M output. Those are useful quality and latency signals, but there is no first-party short-title benchmark showing that the approximately 10× per-title premium over the two nano/lite candidates improves titles enough to justify choosing it when a cheaper authorized provider is also available. [Anthropic model overview](https://docs.anthropic.com/en/docs/about-claude/models/overview) [Pi Anthropic catalog](https://github.com/badlogic/pi-mono/blob/main/packages/ai/src/providers/anthropic.models.ts)

5. **Approximate request cost strongly favors GPT-5 nano, but all figures are estimates rather than tokenizer measurements.** Assumptions: 1,000 input characters ≈ 250 billable input tokens (4 characters/token); a title of at most seven words ≈ 10 visible output tokens; standard synchronous API rates; no cache discount, batch discount, free tier, tools, or fixed title-instruction overhead; no retries. Calculation is `(250 × input $/M + 10 × output $/M) / 1,000,000`.

   | Exact provider/model | Official input / output per 1M | Approx. one title | Approx. 1,000 titles |
   |---|---:|---:|---:|
   | `openai/gpt-5-nano` | $0.05 / $0.40 | **$0.0000165** | **$0.0165** |
   | `google/gemini-2.5-flash-lite` | $0.10 / $0.40 | $0.0000290 | $0.0290 |
   | `anthropic/claude-haiku-4-5` | $1.00 / $5.00 | $0.0003000 | $0.3000 |

   GPT-5 nano and Gemini 2.5 Flash-Lite are reasoning-capable, and provider billing can include non-visible reasoning/thinking tokens; Google explicitly says thinking tokens are billed as output. Thus the table is a visible-token baseline, not a guaranteed invoice. The implementation should request no/minimal reasoning where the exact model/API supports it and cap visible output. [OpenAI GPT-5 nano model card](https://platform.openai.com/docs/models/gpt-5-nano) [Google pricing](https://ai.google.dev/gemini-api/docs/pricing) [Anthropic model overview](https://docs.anthropic.com/en/docs/about-claude/models/overview)

6. **Preserve identifiers as an exact `(provider, modelId)` pair.** Pi’s registry lookup takes two strings and performs `getModel(provider, modelId)`; its generated catalogs preserve provider-native IDs. The extension should neither normalize IDs nor infer that an identically named model routed through another provider is equivalent. This matters for custom providers, gateways, subscription providers, billing, and the privacy gate. Persist or pass through the exact pair returned by Pi, including punctuation, slashes, colons, aliases, or dated snapshots. [Pi `ModelRegistry.find`](https://github.com/badlogic/pi-mono/blob/main/packages/coding-agent/src/core/model-registry.ts) [Pi generated catalog index](https://github.com/badlogic/pi-mono/blob/main/packages/ai/src/models.generated.ts)

7. **Hardcoding any model ID creates lifecycle and behavior risk.** Pi states that built-in catalogs ship with Pi and may be refreshed/cached, so the catalog changes independently of this extension. OpenAI exposes both an alias (`gpt-5-nano`) and a dated snapshot (`gpt-5-nano-2025-08-07`): the alias reduces immediate breakage but can move behavior, while a snapshot improves consistency but will eventually require lifecycle maintenance. Anthropic similarly documents aliases versus pinned IDs, and the Pi catalog currently includes both `claude-haiku-4-5` and `claude-haiku-4-5-20251001`. A fixed identifier therefore trades behavioral drift for retirement risk; runtime catalog resolution plus an ordered policy is safer. [OpenAI GPT-5 nano snapshots](https://platform.openai.com/docs/models/gpt-5-nano) [Anthropic model IDs/versioning discussion](https://docs.anthropic.com/en/docs/about-claude/models/overview) [Pi provider/catalog documentation](https://github.com/badlogic/pi-mono/blob/main/packages/coding-agent/docs/providers.md)

8. **Recommended default policy.** At title-generation time: (a) read `ModelRegistry.getAvailable()`; (b) select only an exact pair in that returned/authenticated set; (c) prefer `openai/gpt-5-nano`, then `google/gemini-2.5-flash-lite`, then `anthropic/claude-haiku-4-5`; (d) preserve the selected pair exactly; (e) if none is available, use the already authenticated current-session model only with explicit product approval, or skip generation and retain the existing fallback title. Never silently acquire/use a different provider merely because the first-ranked model is cheaper. Revalidate the preference list against the current Pi catalog at release time rather than constructing an unregistered ID.

## Gaps

- None of the first-party sources publishes a benchmark specifically for generating seven-word software-session titles. “Summarization,” “instruction following,” “fastest,” and “near-frontier intelligence” are capability proxies, not direct title-quality measurements.
- No live latency benchmark was run, so only provider-stated relative speed/latency characteristics are compared; network region, account tier, cold starts, and reasoning-token generation can dominate such a tiny request.
- Token counts vary by tokenizer and content. The cost table intentionally uses a transparent 4-characters/token and 10-output-token estimate; production telemetry should record actual usage without retaining prompt text.
- A small privacy-safe evaluation using representative, non-sensitive prompts across only the user’s already-authorized providers would be the next step if title quality must decide between GPT-5 nano and Gemini 2.5 Flash-Lite.

## Concise decision

**No universal fixed model is valid under the auth/privacy requirement. Adopt an availability-aware ordered policy; name `openai/gpt-5-nano` as the preferred candidate only when that exact provider/model is already usable.**
